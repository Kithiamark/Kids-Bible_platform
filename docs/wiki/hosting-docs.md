# Hosting the Code Wiki for Free

You asked for the best place to host this documentation “for free for as long as it is possible”. For a public repository, the most durable, low-maintenance option is GitHub Pages.

## Recommended: GitHub Pages (Public Repo)

Why this is the best default:
- Documentation lives in-repo (reviewable in PRs, versioned with code)
- Pages hosting is free for public repositories
- No external account requirements beyond GitHub
- Very long-lived and widely supported

### Option 1: Serve the Markdown directly (no build)

If you’re happy reading the Markdown on GitHub (not a separate website), you can stop here. The wiki index is:
- [docs/wiki/INDEX.md](file:///c:/Users/USER/Project/Kids-Bible_platform/docs/wiki/INDEX.md)

### Option 2: Publish as a simple static site on GitHub Pages

This repo can publish `docs/` (or `docs/wiki/`) through GitHub Pages. Two typical paths:
- Use GitHub Pages to serve a “docs site” generated by a tool like MkDocs (clean navigation/search)
- Use a lightweight static site generator that turns Markdown into HTML (also supports navigation)

If you want the simplest “no new tooling” path, keep the wiki in Markdown and read it on GitHub. If you want a better browsing experience, MkDocs is usually the cleanest upgrade.

## Alternative: Cloudflare Pages

Why it’s good:
- Free tier for public static sites
- Fast global CDN
- Works well if you want to deploy docs from a branch/folder

Trade-off:
- Adds a second platform (Cloudflare) to keep configured

## Alternative: Read the Docs

Best when:
- You adopt a docs build tool (MkDocs or Sphinx)
- You want an established docs hosting UX (versions, search, theme)

Trade-off:
- Usually expects a docs build configuration and/or dependency setup

## Practical Recommendation for This Repo

- Keep the wiki as Markdown in `docs/wiki/` (versioned with the code)
- If/when you want a polished docs website, add MkDocs and publish it via GitHub Pages
