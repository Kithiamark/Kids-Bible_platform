# Beginner Code Roadmap

This folder is a beginner-first guide for learning how to write code clearly, deliberately, and in a way that actually works.

It is written for someone who:
- is still learning the basics
- wants to stop guessing
- wants to understand where code should go
- wants to learn how to test and debug properly
- does not want to rely on “vibe coding”

## What This Is

This is not a theory-heavy course.

It is a practical roadmap that teaches you how to:
- read code without panic
- break problems into steps
- write smaller and clearer functions
- place code in the right file
- debug errors without randomly changing things
- write useful test cases
- build small features with intention

## What “Clear Code” Means

Clear code is code that is:
- easy to find
- easy to read
- easy to change
- hard to misuse

Clear code is usually:
- named well
- split into small pieces
- organized by responsibility
- tested where it matters

## The Core Rule

Do not ask:
- "How do I make this look advanced?"

Ask:
- "If I come back in two weeks, will I understand this quickly?"
- "If something breaks, will I know where to look?"
- "Does this function do one job?"
- "Did I prove it works, or did I only hope it works?"

## How To Study Without Getting Lost

Use this simple cycle every day:

1. Read one small section.
2. Look at one real file in this repo.
3. Explain it in your own words.
4. Write one tiny piece of code.
5. Run it.
6. Fix what breaks.
7. Write down what you learned.

## Golden Beginner Rules

- Small steps beat big motivation.
- Understanding beats speed.
- Working code beats clever code.
- Clear code beats impressive code.
- Debugging is part of coding, not proof that you are bad at it.
- Tests are how you prove something works more than once.

## What Success Looks Like

By the end of this roadmap, you should be able to:
- read a file and explain its purpose
- decide where new code belongs
- write a small feature without mixing responsibilities
- read an error and investigate calmly
- create useful test cases for the most important behavior
