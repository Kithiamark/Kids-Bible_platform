# Checklist: Where Should This Go?

- Is this about HTTP request/response handling? Put it in `api`.
- Is this a business rule or multi-step logic? Put it in `services`.
- Is this database shape or relationship data? Put it in `models`.
- Is this request/response validation? Put it in `schemas`.
- Is this a route screen? Put it in `pages`.
- Is this reusable UI? Put it in `components`.
- Is this shared frontend state? Put it in `store`.
- Is this shared helper logic? Put it in `lib` or `core`.
